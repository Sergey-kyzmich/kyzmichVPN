# kyzmichVPN — Telegram-бот для платных VPN-подписок (3x-ui + ЮKassa)

Python Telegram-бот для продажи VPN-подписок. Когда пользователь оплачивает через **ЮKassa**, бот автоматически создаёт клиента в панели **3x-ui / x-ui** и отправляет данные для подключения (ссылка-подписка + QR + прямая ссылка). Администраторы получают полный контроль над профилями пользователей и могут перенести **всех** пользователей на новое подключение одним действием с уведомлением каждому.

Это Python-реализация, вдохновлённая Go-ботом панели в `test_from_go/` (встроенный Telegram-бот `3x-ui`). В отличие от него, этот бот работает **вне панели** и общается с ней через HTTP API, поэтому его можно запускать где угодно.

---

## Возможности

- 💳 Платные подписки через ЮKassa (карта, СБП, ЮMoney и др. — по настройкам магазина).
- 🤖 Автоматическая выдача доступа: после оплаты создаётся клиент в 3x-ui, и пользователь сразу получает URL подписки, прямую ссылку `vless://` и QR-код.
- 🔁 Продление увеличивает срок действия существующего клиента.
- 👤 «Моя подписка»: статус, трафик, срок действия, QR, продление.
- 🛠 Админ-панель:
  - Статистика (пользователи, активные подписки, онлайн, выручка).
  - Полное управление пользователями: просмотр профиля, включение/отключение, продление, сброс трафика, показ ссылок, сообщение пользователю, удаление.
  - 📣 Рассылка сообщения всем.
  - 🚚 Перенос **всех активных пользователей на новый инбаунд** (новое подключение) одним действием с уведомлением каждого новыми данными.

## Структура проекта

```
run.py                 # точка входа (long polling + фоновая проверка платежей)
config.example.ini     # шаблон конфигурации -> скопировать в config.ini
requirements.txt
bot/
  config.py            # загрузчик конфигурации
  core.py              # экземпляры бота и API панели
  db.py                # SQLite (пользователи, подписки, платежи)
  xui.py               # HTTP API клиент 3x-ui
  links.py             # URL подписки / прямая vless-ссылка / QR
  payments.py          # ЮKassa API + проверка статуса платежа
  provisioning.py      # создание / продление / перенос клиентов
  texts.py             # тексты интерфейса (русский) + форматирование
  keyboards.py         # inline- и reply-клавиатуры
  handlers_user.py     # команды и callback'и пользователя
  handlers_admin.py    # команды и callback'и администратора
```

## Установка

1. **Установите зависимости** (Python 3.10+):

   ```bash
   pip install -r requirements.txt
   ```

2. **Создайте конфигурацию:**

   ```bash
   copy config.example.ini config.ini   # Windows
   # cp config.example.ini config.ini   # Linux/macOS
   ```

   Затем заполните `config.ini` (подробности ниже).

3. **Запуск:**

   ```bash
   python run.py
   ```

## Где взять каждое значение в `config.ini`

### `[telegram]`
| Ключ | Где взять |
| --- | --- |
| `token` | Создайте бота через [@BotFather](https://t.me/BotFather) → `/newbot` → скопируйте токен. |
| `admins` | Ваш числовой Telegram ID. Отправьте `/id` этому боту или узнайте у [@userinfobot](https://t.me/userinfobot). Несколько админов — через запятую. |
| `proxy` | Необязательный SOCKS5-прокси, например `socks5://user:pass@host:1080`. Оставьте пустым, если не нужен. |

### `[xui]` (ваша панель 3x-ui / x-ui)
| Ключ | Где взять |
| --- | --- |
| `base_url` | Адрес панели **включая секретный базовый путь**, без `/panel`. Пример: `https://example.com:2053/SeCrEtPaTh`. Этот путь задаётся при установке 3x-ui (панель → Settings → «Panel URL Path»). |
| `username` / `password` | Логин и пароль панели (панель → Settings → «Security»). Это не SSH-логин. |
| `inbound_id` | ID инбаунда, в который добавляются новые клиенты. Откройте список **Inbounds** в панели (ID указан там) или запустите бота и откройте `🛠 Админ-панель → 📡 Инбаунды`. |
| `flow` | Для **VLESS Reality** укажите `xtls-rprx-vision`. Для обычного TLS/WS/gRPC или VMESS/Trojan/Shadowsocks оставьте **пустым**. |
| `sub_base_url` | URL подписки из настроек панели: панель → Settings → **Subscription** → включите и скопируйте публичный URL (например `https://example.com:2096/sub`). Бот добавляет `/<subId>`. |

> Бот должен иметь доступ к `base_url` с того хоста, где он запущен. Если панель использует самоподписанный сертификат, проверка TLS по умолчанию отключена.

### `[yookassa]`
| Ключ | Где взять |
| --- | --- |
| `shop_id` | Идентификатор магазина в личном кабинете ЮKassa: [Интеграция → Ключи API](https://yookassa.ru/my/merchant/integration/api-keys). |
| `secret_key` | Секретный ключ API на той же странице. Создайте новый ключ, если его ещё нет. |
| `return_url` | URL, куда пользователь вернётся после оплаты. Можно указать ссылку на бота, например `https://t.me/YourBotUsername`. |

#### Подключение ЮKassa

1. Зарегистрируйте магазин на <https://yookassa.ru/> и пройдите модерацию.
2. В личном кабинете откройте **Интеграция → Ключи API** и скопируйте `shopId` и `Секретный ключ`.
3. Укажите их в `config.ini` в секции `[yookassa]`.
4. Валюта платежей берётся из `[subscription] currency` (обычно `RUB`).

> Webhook не обязателен: бот проверяет статус платежа через API (кнопка «Я оплатил» + фоновый опрос каждые 30 секунд).

### `[subscription]` и `[plans]`
- `default_gb` / `limit_ip`: значения по умолчанию для новых клиентов (0 = безлимит).
- `currency`: валюта платежей и отображения (например `RUB`). Должна совпадать с валютой магазина в ЮKassa.
- `[plans]`: каждая строка — `ключ = дни:цена:гб`. Тарифы можно добавлять и удалять; меню покупки формируется из этого раздела.

## Как работает оплата → доступ

1. Пользователь выбирает тариф → бот создаёт платёж в ЮKassa и отправляет ссылку на оплату.
2. Пользователь оплачивает. Бот подтверждает оплату двумя способами:
   - пользователь нажимает **«✅ Я оплатил — проверить»**, или
   - фоновый процесс опрашивает статус платежа каждые 30 секунд.
3. После подтверждения (`status = succeeded`) бот создаёт/продлевает клиента в 3x-ui и отправляет данные для подключения. Выдача доступа идемпотентна (один платёж применяется только один раз).

## Замечания и рекомендации

- Запускайте бота на хосте с доступом к панели (часто это сам VPN-сервер).
- Не публикуйте `config.ini` — в нём логин панели и ключи ЮKassa.
- Для продакшена используйте менеджер процессов (systemd / `pm2` / Docker), чтобы бот перезапускался автоматически.
- URL подписки всегда корректен; прямая ссылка `vless://` собирается по возможности из настроек инбаунда (VLESS TCP/Reality/TLS/WS/gRPC).
